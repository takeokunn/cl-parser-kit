# Code of Conduct

`cl-parser-kit` expects respectful, technically focused collaboration.

## Our Standards

- critique code and behavior, not people
- prefer concrete evidence over rhetoric
- keep review comments specific and actionable
- assume good intent, but do not hand-wave away reproducible problems
- avoid harassment, discrimination, or personal attacks

## Scope

This policy applies to repository discussions, issues, pull requests, and
other project communication channels.

## Reporting

For private conduct reports, use the contact path documented in `SECURITY.md`.
If the report is not security-sensitive, say that it is a conduct concern so it
can be routed correctly.

## Enforcement

Maintainers may edit, hide, lock, or remove comments and contributions that do
not meet this policy. Repeated or severe violations can lead to temporary or
permanent exclusion from project spaces.

## Project Fit

This repository is intentionally small and technical. Direct, rigorous review
is welcome; hostility is not.
